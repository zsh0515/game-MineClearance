# 扫雷游戏开发计划

## 一、项目概述

**项目名称**：经典扫雷（Minesweeper）  
**项目类型**：休闲益智游戏  
**核心玩法**：在雷区中揭开方块，避免触雷，找出所有非雷方块即胜利  
**目标用户**：休闲游戏玩家，适合各年龄段

---

## 二、技术选型

### 推荐方案：HTML5 + CSS3 + JavaScript（原生）

**优势**：
- 无需构建工具，直接浏览器运行
- 学习曲线平缓，适合教学
- 部署简单，可发布到任意静态托管平台

**备选方案**：
- Unity 2D：适合想学习游戏引擎的开发者
- Python + Pygame：适合有Python基础的开发者
- React/Vue：适合想学习前端框架的开发者

## 三、游戏设计规格

### 3.1 难度等级

| 等级 | 行数 | 列数 | 地雷数 | 格子总数 | 雷占比 |
|------|------|------|--------|----------|--------|
| 初级 | 9 | 9 | 10 | 81 | 12.3% |
| 中级 | 16 | 16 | 40 | 256 | 15.6% |
| 高级 | 16 | 30 | 99 | 480 | 20.6% |

### 3.2 核心游戏机制

#### 3.2.1 格子状态
- **未揭开**：初始状态，显示为凸起的方块
- **已揭开**：显示数字（0-8）或空白
- **标记地雷**：玩家认为有雷的格子，显示旗帜图标
- **问号标记**：玩家不确定的格子，显示问号
- **已爆炸**：游戏结束时显示触发的地雷

#### 3.2.2 数字含义
- **1**：周围8格中有1个地雷
- **2**：周围8格中有2个地雷
- **...**
- **8**：周围8格中有8个地雷
- **0**：周围无地雷，自动扩展揭开相邻格子

#### 3.2.3 胜利/失败条件
- **胜利**：揭开所有非雷格子
- **失败**：揭开地雷

```
├─────────────────────────────────────┤
│            状态栏/提示               │

```

#### 3.3.2 配色方案
- 未揭开格子：浅灰色 (#C0C0C0)
- 已揭开格子：深灰色 (#E0E0E0)
- 数字颜色：
  - 1：蓝色 (#0000FF)
  - 2：绿色 (#008000)
  - 3：红色 (#FF0000)
  - 4：深蓝色 (#000080)
  - 5：棕色 (#800000)
  - 6：青色 (#008080)
  - 7：黑色 (#000000)
  - 8：灰色 (#808080)
- 地雷：红色背景 (#FF0000)
- 旗帜：红色旗帜图标

### 3.4 用户交互设计

#### 3.4.1 鼠标操作
- **左键单击**：揭开格子
- **右键单击**：标记/取消标记地雷
- **双击（左键）**：当已揭开的数字周围标记数等于数字时，自动揭开周围未揭开格子

#### 3.4.2 第一点击安全机制
- 第一次点击时生成雷区，确保第一次点击不会触雷
- 第二次点击后才生成雷区，第一次点击位置及其周围保证安全

## 五、代码架构设计

### 5.1 数据结构

```javascript
// 游戏状态
const gameState = {
  status: 'waiting', // waiting | playing | won | lost
  rows: 9,
  cols: 9,
  mines: 10,
  board: [], // 二维数组存储每个格子的状态
  startTime: null,
  flagCount: 0
};

// 格子状态
const cell = {
  isMine: false,      // 是否是地雷
  isRevealed: false,  // 是否已揭开
  isFlagged: false,   // 是否被标记
  isQuestion: false, // 是否是问号
  adjacentMines: 0    // 周围雷数
};
```

### 5.2 核心算法

1. **雷区生成**：Fisher-Yates 洗牌算法随机放置地雷
2. **数字计算**：遍历每个格子，统计周围8格的地雷数
3. **空白扩展**：BFS 广度优先搜索，揭开所有相邻的空白格子
4. **胜负判定**：
   - 胜利：已揭开格子数 = 总格子数 - 地雷数
   - 失败：揭开的格子是地雷
